# 🍷 心动的生活 - 消除类游戏

以微信小程序游戏《心动的生活》为灵感，在其核心消除玩法基础上加入了**目标酒杯加分**、**Combo 连击**、**单轮额外奖励**、**难度梯度**、**障碍石**等创新机制。使用纯 TypeScript + HTML5 Canvas 实现，支持跨平台运行。

## 📖 游戏规则

### 基本玩法

- 游戏棋盘为 **6 行 × 4 列** 的网格（可配置）
- 每轮发放 **3 个盘子** 到临时摆放区
- 每个盘子有 **6 个酒杯槽位**，新盘子随机填充 1~3 个酒杯（可配置）
- 玩家按任意顺序将盘子放置到棋盘的空格子中（障碍石格子不可放置）

### 合并机制（吸收模型）

- 盘子放置后，自动检查上下左右四个方向的相邻盘子
- 若与任意相邻盘子存在**同类型酒杯**，则通过**类型并集匹配**构建**参与组**（仅一层邻居，不做 BFS 扩展）
- 参与组内的盘子按**放置时间从早到晚**排序，依次执行**吸收**：
  - 每个盘子从自己持有的共享类型中，选择**总数量最多**的类型进行吸收
  - 吸收时清空自身，非吸收类型挤出到**剩余池**
  - 从参与组内所有盘子和剩余池中收集该类型的所有酒杯（**上限 6 个**）
  - 超出 6 个的溢出部分放入剩余池
- 剩余池中的酒杯按数量降序，填入参与组内**空位最多**的盘子
- 空盘自动消除，6 个同类型满盘消除并得分
- 接收了剩余分配且未被消除的盘子作为触发点，递归执行合并（**连锁反应**），直到棋盘达到**稳定态**
- 若新盘子与所有相邻盘子**没有任何共同类型**，则不触发合并
- 详细算法文档见 [docs/merge-algorithm.md](docs/merge-algorithm.md)

### 消除规则

- **空盘消除**：盘子中酒杯数量为 0 时自动消除，不得分
- **满盘消除**：盘子装满 6 个相同类型酒杯时消除并得分

### 得分系统

| 机制 | 规则 |
|------|------|
| Combo 连击 | 单次移动中第 N 个满盘消除得 N 分 |
| 目标酒杯 | 消除目标酒杯类型的满盘时得分 ×2 |
| 单轮奖励 | 一轮中消除满盘达到阈值时获得额外奖励（默认：3个→+1分，6个→+5分，9个→+10分） |
| 障碍石破除 | 障碍石封印完全破除时得 初始封印层数 × 3 分 |

### 目标酒杯

- 游戏开始时随机选择 2 种目标酒杯类型（可配置）
- 初始刷新阈值为 4 次满盘消除，每次刷新后阈值增长 2（即 4, 6, 8, 10, ...）
- 刷新时同时解锁 1 种新酒杯类型（直到达到上限 8 种）
- 刷新后在空格中随机生成新障碍石（最多 2 个，全场上限 6 个）
- 画面上方显示目标酒杯刷新进度条

### 难度梯度

- 游戏开始时只有 **3 种**酒杯类型
- 每次目标酒杯刷新时，解锁 1 种新酒杯类型（直到达到上限 **8 种**）
- 酒杯类型渐进增加，前期容易上手，后期难度逐步提升

### 障碍石

- 游戏开始时随机放置 **4 个**障碍石（可配置）
- 盘子不能放在障碍石占据的格子上
- 每个障碍石有 **1~3 层封印**
- 相邻位置（上下左右）完成满6消除时，破除 1 层封印
- 封印完全破除后：
  - 获得 初始封印层数 × 3 分
  - 获得当前目标酒杯刷新阈值 50% 的进度（溢出累计到下轮）
  - 障碍石从棋盘移除，格子变为可放置状态
- 目标酒杯刷新后，在最多 2 个空格中随机刷出新障碍石（全场上限 6 个）
- 障碍石外观按封印层数区分：1 层石质、2 层铁质、3 层钻石质

### 游戏结束

- 当棋盘所有可放置格子（非障碍石）都被盘子占据时，游戏结束
- 即使临时区仍有未放置的盘子，只要棋盘无空位即判定结束

## ⌨️ 快捷键

| 按键 | 功能 |
|------|------|
| `E` | 进入/退出编辑模式 |
| `Enter` | 退出编辑模式并加载编辑后的状态 |
| `0`~`7` | 编辑模式下向选中盘子添加对应类型酒杯 |

## � 调试工具

### 编辑模式

按 `E` 键进入编辑模式，可手动编辑棋盘和临时区的盘子内容，用于测试和调试特定场景。点击格子选中后用数字键添加酒杯，退出编辑模式后以编辑后的状态开始游戏。

### 棋盘快照日志

按 `D` 键开启，每次放置操作自动输出完整棋盘状态快照到日志。开启时画面右上角显示红色 "DUMP" 指示器。

### 事件日志系统

游戏自动记录所有事件（放置、合并、消除、得分、轮次、障碍石等），按 `L` 键可下载日志文件。控制台可通过 `window.gameLogger` 访问日志对象。

## 🏗 架构设计

项目采用**逻辑层与渲染层彻底分离**的架构：

```
┌─────────────────────────────────────────┐
│              表现层                      │
│  CanvasRenderer  │  InputHandler        │
├─────────────────────────────────────────┤
│            抽象接口层                    │
│    IRenderer     │  IInputHandler       │
├─────────────────────────────────────────┤
│             游戏逻辑层                   │
│  GameEngine → BoardState                │
│             → MergeAlgorithm            │
│             → ScoreCalculator           │
│             → PlateGenerator            │
│             → GameConfig                │
└─────────────────────────────────────────┘
```

游戏逻辑层不依赖任何 DOM、Canvas 或平台特定 API，未来扩展到微信小程序、QQ 小程序、支付宝小程序等平台时，只需实现对应的渲染适配器，无需修改游戏逻辑层。

## ⚙️ 可配置参数

所有关键游戏参数均可配置，方便后续调整难度和平衡性：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `boardRows` | 6 | 棋盘行数 |
| `boardCols` | 4 | 棋盘列数 |
| `glassTypeCount` | 8 | 酒杯种类数量上限 |
| `initialGlassTypes` | 3 | 初始解锁酒杯种类数 |
| `minGlassesPerPlate` | 1 | 新盘子最少酒杯数 |
| `maxGlassesPerPlate` | 3 | 新盘子最多酒杯数 |
| `slotsPerPlate` | 6 | 每个盘子槽位数 |
| `platesPerRound` | 3 | 每轮发放盘子数 |
| `targetGlassCount` | 2 | 目标酒杯数量 |
| `targetGlassRefreshThreshold` | 4 | 初始目标酒杯刷新阈值（累计满盘消除数） |
| `targetRefreshGrowth` | 2 | 每次刷新后阈值增长量 |
| `initialObstacles` | 4 | 初始障碍石数量 |
| `maxObstacles` | 6 | 全场障碍石上限 |
| `obstaclesPerRefresh` | 2 | 目标刷新后新增障碍石上限 |
| `roundBonuses` | `[{3,1},{6,5},{9,10}]` | 单轮额外奖励（阈值→奖励分数） |

配置校验规则：
- 参数不合法时自动回退到默认值并在控制台输出警告
- 例如 `minGlassesPerPlate > maxGlassesPerPlate` 时两者均回退默认值

自定义配置示例（在 `src/main.ts` 中修改）：

```typescript
const engine = new GameEngine({
  glassTypeCount: 6,           // 酒杯种类上限
  initialGlassTypes: 2,        // 初始只解锁 2 种，更容易上手
  maxGlassesPerPlate: 3,       // 新盘子最多酒杯数
  targetGlassCount: 3,         // 增加目标酒杯数量
  targetGlassRefreshThreshold: 3, // 更快刷新目标酒杯
  targetRefreshGrowth: 1,      // 阈值增长更慢
  initialObstacles: 2,         // 减少初始障碍石
  maxObstacles: 4,             // 降低障碍石上限
  roundBonuses: [
    { threshold: 2, bonus: 2 },
    { threshold: 5, bonus: 8 },
  ],
});
```

## 📱 跨平台部署

### Web 浏览器

直接 `npm run build` 后部署 `dist/` 目录到任意静态服务器即可。支持桌面和移动端浏览器。

### Android

```bash
# 1. 构建 Web 产物
npm run build

# 2. 添加 Android 平台（首次）
npm run cap:add:android

# 3. 同步 Web 产物到原生项目
npm run cap:sync

# 4. 在 Android Studio 中打开项目
npm run cap:open:android
```

### iOS

```bash
# 1. 构建 Web 产物
npm run build

# 2. 添加 iOS 平台（首次）
npm run cap:add:ios

# 3. 同步 Web 产物到原生项目
npm run cap:sync

# 4. 在 Xcode 中打开项目
npm run cap:open:ios
```

> iOS 打包需要 macOS 环境和 Xcode。

### 小程序平台（规划中）

架构已预留渲染适配器扩展点。未来接入微信/QQ/支付宝小程序时，只需：

1. 实现对应平台的 `IRenderer` 渲染适配器
2. 实现对应平台的 `IInputHandler` 输入适配器
3. 游戏逻辑层代码无需任何修改
