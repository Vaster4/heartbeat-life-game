# Feature #002: 参与组重分配合并算法

## 状态：已完成 ✅（算法已重写为定向转移策略）

## Commit: `54203f1`（初版），`0f9018e`（重写为定向转移）

## 需求描述

重写合并算法，将旧的"逐类型连通合并 + 位移"策略替换为"参与组重分配"策略。新盘子放入后，如果与相邻盘子有共同酒杯类型，则将该盘子和所有相邻盘子组成参与组，把所有酒杯按类型归类，每种类型集中到一个盘子上，多余盘子被消除。

## 算法规则（v2 - 定向转移策略）

1. 放入新盘子后，检查它与所有相邻盘子是否有共同酒杯类型
2. 如果没有任何共同类型 → 不合并
3. 如果有 → 通过 BFS 扩展，将所有通过直接相邻共同类型连通的盘子纳入同一参与组
4. 收集组内所有相邻盘子对，按时间戳差从大到小排序
5. 对每对相邻盘子（新→老方向）：
   - 共同类型的酒杯从新盘子转移到老盘子（不超过6上限）
   - 老盘子上的非共同类型酒杯推回给新盘子（不超过6上限）
6. 每个盘子趋向单一类型化，合并后盘子上的类型是合并前就有的
7. 空盘消除，6个同类型满盘消除
8. 消除后继续迭代直到棋盘稳定

## 与旧算法的区别

| 对比项 | 旧算法（v1 全局重分配） | 新算法（v2 定向转移） |
|--------|--------|--------|
| 合并范围 | 参与组内所有酒杯统一重分配 | 逐对相邻盘子定向转移 |
| 分配策略 | 每种类型集中到一个盘子，按优先级分配 | 共同类型从新→老转移，非共同类型从老→新回推 |
| 类型归属 | 合并后盘子可能出现原本没有的类型 | 合并后盘子上的类型一定是合并前就有的 |
| 盘子消除 | 多余盘子直接变空消除 | 空盘和满盘消除 |
