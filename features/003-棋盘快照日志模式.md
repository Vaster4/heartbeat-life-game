# Feature #003: 棋盘快照日志模式

## Commit: 待回填

## 背景

原有日志系统只记录事件（放置、合并、消除），无法直接还原每一步的棋盘状态，
调试合并算法问题时需要手动追踪所有操作来推算棋盘状态，非常容易出错。

## 功能描述

新增棋盘快照（dump）日志模式，开启后在每次放置操作时自动输出完整棋盘状态：

- 放置后/合并前：输出放置盘子后、执行合并算法前的棋盘快照
- 合并/消除后：如果发生了合并或消除，输出最终棋盘快照

### 快照格式

```
[DEBUG] [DUMP] --- 放置后/合并前 ---
  (0,1): plate-1 [3,3] ts=1
  (2,2): plate-3 [7,7,7,7] ts=3
  ...
```

每行一个非空格子，包含盘子 ID、酒杯列表、放置时间戳。

### 使用方式

- 游戏中按 D 键切换开关（默认关闭）
- 控制台：`gameLogger.dumpEnabled = true`
- 代码：`engine.logger.dumpEnabled = true`
- 按 L 键下载日志文件时包含快照数据

### 视觉反馈

开启 dump 模式后，画面右上角显示红色圆点 + "DUMP" 文字，提示用户当前处于快照记录状态。
再次按 D 键关闭后指示器消失。
